# Platform and Integration Compatibility Contracts

Baseline: v1.10.0, commit `165ec46`. Chrome, Edge, and Firefox are supported extension targets. Android work in this refactor is limited to origin security and shared-contract hardening; the Android wrapper is not required to reach browser feature parity.

## T1.5 Freeze Requirements

- Assert three generated manifests, permission/capability alignment, stable browser/Firefox identity, and explicit unsupported degradation.
- Fixture DeepSeek route matching, request/body preservation, SSE historical shapes, malformed event visibility, and official API abort/error behavior.
- Fixture every known MCP v1 config/transport/protocol path and expose current unknown-version/transport behavior as a gap; later hardening must reject before network or write.
- Compare the complete Shell TypeScript/host tool catalogs and schemas, and exercise invalid/future Native envelopes.
- Add Android origin and structured-command negative tests when a JDK/Gradle-capable environment is available; until then, do not claim Android runtime compatibility is proven.

## Current Evidence Limit

The platform/integration review ran 15 related Vitest files / 106 tests with a hard 60-second timeout, passed manifest policy, MCP smoke, and all 17 Shell smoke calls, and left no Vitest/smoke child process. Android could not be built or run because JDK/Gradle was unavailable; route matchers, MCP future-version behavior, full host catalog/schema parity, and Android origin/bridge negatives remain T1.5 fixture work.
